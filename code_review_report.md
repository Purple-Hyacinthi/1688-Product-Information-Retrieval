### 🔴 关键问题（必须修复）
- [src/main_window.py:120] 空的 except 语句捕获所有异常
  - **规则:** PP-72 (保持简单并最小化攻击面), CC-45 (使用异常而非返回码)
  - **原则:** 空的异常处理会隐藏错误和安全漏洞，包括 KeyboardInterrupt 和 SystemExit
  - **建议:** 至少记录异常或捕获特定异常类型：`except Exception as e: logger.error(...)`

- [tests/test_main_window.py] 测试覆盖率严重不足
  - **规则:** PP-91 (测试你的软件), CC-194 (测试)
  - **原则:** L4 级别要求 80% 测试覆盖率，当前只有一个构造函数测试，缺少搜索、代理分析、UI 交互等关键功能测试
  - **建议:** 添加单元测试覆盖主要业务逻辑（搜索、分析、错误处理），集成测试验证组件协作

### 🟡 重要问题（应该修复）
- [src/main_window.py:23-107] _setup_ui 函数过长（约80行逻辑行）
  - **规则:** CC-20 (函数应短小)
  - **原则:** L4 级别函数长度阈值 ≤30 行，长函数难以理解、测试和维护
  - **建议:** 提取子函数：_create_search_area(), _create_result_area(), _create_agent_area(), _create_status_bar()

- [src/main_window.py:8] __init__ 方法参数过多（5个）
  - **规则:** CC-26 (函数参数), CC-147 (参数过多)
  - **原则:** L4 级别参数阈值 ≤3 个，过多参数增加认知负担和测试复杂度
  - **建议:** 使用参数对象或依赖注入容器，或将相关参数分组为配置对象

- [src/main_window.py] MainWindow 类违反单一职责原则
  - **规则:** CA-8 (单一职责原则)
  - **原则:** 类同时负责 UI 布局、搜索业务逻辑、代理分析、配置管理，变更风险高
  - **建议:** 提取 SearchController、AgentController 等业务类，MainWindow 只负责 UI 协调

- [src/main_window.py:141,192] 重复的错误处理模式
  - **规则:** PP-15 (DRY), CC-37 (不要重复自己)
  - **原则:** L4 级别允许最多2次重复，这是第2次出现，建议提取避免未来重复
  - **建议:** 提取通用函数 `_handle_async_error(callback, error_message)` 处理线程错误
